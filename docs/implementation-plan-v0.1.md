# KodeXimi v0.1 Implementation Plan

Implementation proceeds in small committed slices.

## Phase 1: Python runtime skeleton

- Add `pyproject.toml`.
- Add `src/kodeximi`.
- Implement `init`, `doctor`, `session start/status/stop`.
- Implement TaskSpec validation.
- Implement SQLite schema.

## Phase 2: Snapshot and fake job loop

- Enforce single-root lock.
- Require git-backed direct mode.
- Record pre/post git status.
- Generate patch and changed-files evidence.
- Run fake transport to validate state flow.

## Phase 3: Verification and evidence

- Implement argv-only verification runner.
- Generate runtime `VERIFY.md` and `verify.json`.
- Generate `EVIDENCE_DIGEST.md`.
- Record observed usage with data-completeness status.

## Phase 4: RawWire transport

- Add Kimi `--wire` smoke scripts.
- Add `kodeximi doctor --wire-smoke`.
- Use Kimi Wire's snake_case initialize fields: `protocol_version`, `client`, and `capabilities`.
- Use a 15-second initialize timeout; local Kimi 1.44.0 can take roughly 7 seconds to answer initialize.
- Treat repeated `WIRE_INIT_TIMEOUT` as a hard integration blocker for real Kimi transport, not as a reason to fallback to `--print`.
- Add WorkerEvent adapter.
- Add approval dispatcher.
- Add minimal `kimi-wire` transport only after initialize smoke passes. It must archive `wire.jsonl`, reject Shell approvals, and require `RESULT.md`.
- Reject Kimi Shell.
- Approve/reject writes by PathPolicy.
- Persist `wire.jsonl` and approval decisions.

## Phase 5: Review and rework

- Record Codex review decisions.
- Keep `rework` as a recorded decision; do not automatically rerun.
- Generate `REWORK_BRIEF.md` before a second attempt.
- Enforce `max_attempts`.
- Provide `review package`, `job rerun`, and `job cancel` CLI surfaces.

## Phase 6: Codex plugin-lite

- Add repo-local plugin-lite docs.
- Add strict session protocol.
- Add TaskSpec and review checklists for Codex.

## v0.1 acceptance

- `kodeximi init` creates project state.
- `kodeximi doctor` returns JSON.
- `kodeximi session start` enables strict session mode.
- invalid TaskSpec returns structured JSON errors.
- fake job reaches `needs_review`.
- runtime verification is generated by KodeXimi.
- evidence digest exists and stays short.
- review decision is recorded.
- review package returns the latest evidence paths.
- rework rerun creates `REWORK_BRIEF.md`.
- same-root lock prevents concurrent active jobs.
