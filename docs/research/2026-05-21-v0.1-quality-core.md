# 02. KodeXimi v0.1 最小保质量设计

## 2.1 目标

v0.1 是最小可用但保质量的版本，不是玩具 demo。它必须跑通：

```text
Codex -> TaskSpec -> KodeXimi -> Kimi attempt -> RESULT -> runtime VERIFY -> diff -> usage -> Codex review -> accept/rework
```

适用范围：

- 单个 code_edit 需求。
- 多个互不影响的类似需求。
- 本地项目 opt-in。
- Kimi 作为 system CLI worker。

不适用范围：

- 复杂依赖 DAG。
- 强安全隔离任务。
- DB 写入、通知发送、web crawling。
- 模型路由系统。

## 2.2 必须保留的质量核心

### TaskSpec

TaskSpec 是控制面输入。它至少包含：

```json
{
  "version": 1,
  "job_type": "code_edit",
  "goal": "...",
  "files_to_read": ["..."],
  "write_policy": {
    "modify_files": ["..."],
    "create_files": [],
    "delete_files": [],
    "generated_dirs": [],
    "forbidden_paths": [".kodeximi/", ".git/", ".env"]
  },
  "verification": {
    "commands": [
      {
        "id": "pytest_case",
        "class": "python_test",
        "command": "python -m pytest tests/test_x.py",
        "expected_exit_code": 0
      }
    ]
  },
  "success_criteria": ["..."],
  "max_attempts": 2
}
```

### Kimi worker profile

v0.1 不做 Kimi plugin，但必须做 Kimi `--agent-file`：

```text
.kodeximi/agents/kimi-codex-worker.yaml
.kodeximi/agents/kimi-codex-worker-system.md
```

目的：

- 让 Kimi 明确自己是 worker。
- 要求写 RESULT.md。
- 禁止写 TASK、REWORK_LOG、CODEX_REVIEW、SQLite、agent/skill 配置。
- 禁止宣布 accepted。
- 遇到任务边界不足时写 Open Issues。

### RawWireTransport

v0.1 使用 system Kimi：

```text
kimi --wire --agent-file ... --skills-dir ... --work-dir <project-root>
```

原因：此前验证发现 Python SDK 版本落后于 system Kimi，SDK 内置 Kimi CLI / Wire 能力更旧。因此 v0.1 production 走 system Kimi Wire。

### Runtime VERIFY

Kimi 不写正式 `VERIFY.md`。KodeXimi 运行 verification commands 并生成：

- command
- cwd
- exit_code
- duration
- stdout head/tail
- stderr head/tail
- artifact checks
- conclusion: pass / fail / partial

### diff-summary

KodeXimi 生成：

- changed files
- created files
- deleted files
- git diff --stat
- unexpected changes
- forbidden path warning
- short diff excerpts

### usage

第一版记录：

- job_id
- attempt_no
- source
- quality: exact / partial / unavailable
- raw token_usage payload
- duration_seconds
- raw_status_updates_count

### needs_review

Kimi 完成后只能进入 `needs_review`，不能直接完成。

## 2.3 状态机

Job state：

```text
created
running
needs_review
accepted
failed
blocked
cancelled
```

Attempt state：

```text
created
running
worker_finished
verifying
evidence_ready
failed
cancelled
```

采用同一 job 多 attempt：

```text
.kodeximi/tasks/job-001/
  TASK.md
  REWORK_LOG.md
  CODEX_REVIEW.md
  attempts/
    001/
      RESULT.md
      VERIFY.md
      diff-summary.md
      usage.json
    002/
      RESULT.md
      VERIFY.md
      diff-summary.md
      usage.json
```

## 2.4 CLI

```powershell
kodeximi init
kodeximi enable
kodeximi disable
kodeximi doctor --json
kodeximi job run --from-json task-spec.json --wait --json
kodeximi job status job-001 --json
kodeximi job status --pending --summary
kodeximi review decide job-001 --decision accepted --reason-file review.md --json
kodeximi usage job-001 --summary
```

## 2.5 SQLite

Tables:

- jobs
- attempts
- reviews
- usage
- events

Raw wire logs 写到文件，不塞进 SQLite 主表：

```text
.kodeximi/logs/job-001-attempt-001-wire.jsonl
```

## 2.6 并行能力

v0.1 支持多个互不影响任务并行，判断标准：

- modify_files 不重叠。
- create_files 不重叠。
- generated_dirs 不重叠。
- 没有 depends_on。

命令示意：

```powershell
kodeximi batch run --from-json batch.json --max-parallel 3 --wait --json
```

冲突时拒绝并行或要求串行。

## 2.7 不做完整 plugin/hook 的边界

v0.1 可以不做：

- Codex plugin
- Codex hook
- Kimi plugin
- Kimi extra hook

但必须做：

- Kimi agent-file
- controlled skills-dir
- RawWire approval/boundary
- control-plane protection
- runtime VERIFY
- diff-summary
- usage

判断：Codex-side plugin/hook 不直接影响一次已明确触发任务的执行质量，但影响长期触发可靠性和防绕过能力。完整产品需要。
