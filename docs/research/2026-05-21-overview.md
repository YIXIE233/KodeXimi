## 2. KodeXimi 的一句话定义

KodeXimi 是一个 **Codex-first 的本地委托执行 runtime**。

```text
Codex 定义任务和验收标准；
Kimi CLI 执行一个 bounded attempt；
KodeXimi 记录、验证、打包 evidence，并统计 usage；
Codex 基于 evidence 做 review 和 rework 决策。
```

## 3. 它不是

KodeXimi 不是：

```text
多 agent OS；
强安全沙盒；
Kimi-first runtime；
模型路由系统；
dashboard-first 产品；
通用任务平台。
```

## 4. 当前第一版共识

v0.1 应该是：

```text
code_edit only；
local CLI runtime；
project opt-in；
system Kimi CLI --wire；
Kimi bounded attempt；
runtime-generated VERIFY；
runtime-generated diff-summary；
per-attempt usage；
needs_review；
Codex review decide；
same-job multiple attempts；
basic independent parallelism。
```

## 5. 为什么需要 KodeXimi

Codex 适合：

```text
理解用户意图；
定义边界；
审查结果；
决定是否接受；
向用户解释。
```

Kimi CLI 适合：

```text
长上下文读取；
局部代码/脚本修改；
批量文件处理；
命令行环境中的执行工作。
```

两者之间需要一个稳定协议，否则会出现：

```text
任务边界不清；
Kimi 自称完成但未必满足需求；
Codex 难以知道实际改了什么；
验证记录可能只是 worker 自述；
返工没有结构；
usage 不可统计。
```

KodeXimi 的价值是把委托变成可记录、可验证、可审查、可返工、可计量的事务。
