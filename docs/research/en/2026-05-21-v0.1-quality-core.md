# 02. KodeXimi v0.1 Quality Core

## Goal

KodeXimi v0.1 should prove one workflow:

```text
Codex -> TaskSpec -> KodeXimi -> Kimi bounded attempt -> RESULT -> runtime VERIFY -> diff-summary -> usage -> Codex review -> accept/rework
```

## Required components

- Project-scoped CLI runtime.
- `code_edit` only.
- `TaskSpec` with goal, read files, write policy, verification commands, success criteria, max attempts.
- Kimi worker profile via `--agent-file`.
- Controlled `--skills-dir`.
- RawWireTransport over system `kimi --wire`.
- Kimi writes `RESULT.md`.
- KodeXimi writes `VERIFY.md`, `diff-summary.md`, `usage.json`.
- SQLite job/attempt/review/usage state.
- `needs_review` gate.
- Same-job multiple attempts for rework.
- Basic independent parallelism via disjoint write sets.

## Not required in v0.1

- Codex plugin.
- Codex hook.
- Kimi plugin.
- Extra Kimi hook.
- Dashboard.
- Materialized workspace.
- Full DAG scheduler.
- Step-level routing.

Codex-side plugin/hook improves long-term workflow reliability but is not required for the quality of one explicitly triggered delegation. Kimi agent-file and RawWire boundary are required.
