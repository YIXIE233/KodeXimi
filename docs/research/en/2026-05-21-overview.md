# 00. Overview

This repository preserves two related but distinct lines of thinking.

## KodeXimi

KodeXimi is a Codex-first local delegation runtime:

```text
Codex defines and reviews.
Kimi CLI executes a bounded attempt.
KodeXimi records, verifies, packages evidence, and tracks usage.
```

It is not a multi-agent OS, not a step-level router, and not a full sandbox product.

## Step-level harness routing

Step-level harness routing is a separate research idea. It asks whether a coding harness execution loop can be decomposed into visible steps such as model calls, tool calls, tool results, context updates, compaction, and final decisions, then assign different models to different model-call steps.

This is not the same as multi-agent collaboration or subtask delegation.

## Final separation

- KodeXimi: cross-harness block-level delegation.
- Step-level routing: intra-harness step-level model assignment.

KodeXimi v0.1 should stay focused on code_edit delegation from Codex to Kimi with runtime-generated verification, diff summaries, usage records, and Codex review.
